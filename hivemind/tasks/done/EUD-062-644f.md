## Description
Live E2E finding (EUD-061, user 2026-06-05): agent codex threads inherit the user's ENTIRE personal codex environment — global config.toml MCP servers (playwright/pencil/node_repl), 4 plugins (browser/documents/spreadsheets/presentations), and 10 personal skills from ~/.codex/skills. The per-thread injection only ADDS eud-tools; it suppresses nothing. The agent must run with ONLY the MCP servers/skills the project configures.

## Investigation grounding (orchestrator, 2026-06-05, codex CLI on this machine)
- `-c mcp_servers={...}` config override replaces the WHOLE table (TOML table override semantics) -> passing the full table with ONLY eud-tools via `CodexConfig.config_overrides` suppresses the personal MCP servers. CONFIRM live that the thread still gets eud-tools through this path (it may interact with the per-thread `thread_start(config=...)` merge).
- `features.plugins=false` (stable flag, `codex features list`) disables plugins.
- Skills have NO global disable (developers.openai.com/codex/skills: only per-skill `[[skills.config]] path/enabled=false`). Skills load from repo `.agents/skills` + `$HOME/.agents/skills` + `$CODEX_HOME/skills` + built-ins.
- `codex exec --ignore-user-config` exists (auth still uses CODEX_HOME) but the SDK spawns `codex app-server`, which does NOT expose that flag. `CodexConfig.launch_args_override` MIGHT allow injecting it if the flag is global — verify; if app-server rejects it, fall back per below.
- Dedicated CODEX_HOME would isolate everything but relocates auth.json (BYO token rotation divergence risk) — last resort only; document the trade-off if chosen.
