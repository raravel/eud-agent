## Context

First live codex session against the full tool surface exposed five failures:

1. `xdat_get` called with invented arg names (`{"table","field","id"}`) →
   "missing required argument 'dat'" repeatedly.
2. `btn_get` → bridge lua crash `bad argument #2 to 'tonumber' (base out of
   range)` at ZZZ_10_agent_bridge.lua:991.
3. `build_errors` → HTTP 500 at /tools/call (shim shows "Server error 500").
4. location_write changeset item rendered rollback bookkeeping as a diff
   (`ori: {mapPath, backupPath} → new: {action, name, locationId}`).
5. Generated code was SCMDraft classic text-trigger syntax
   (`Trigger { players = ..., conditions = ..., actions = ... }`), not epScript.

## Root causes + fixes

1. **mcp_shim never advertised the tool schemas.** FastMCP derives inputSchema
   from the function SIGNATURE; every tool was registered as
   `_tool(args: dict | None)`, so codex saw a single untyped `args` object and
   invented parameter names. → Rewrote the shim on the lowlevel
   `mcp.server.lowlevel.Server`: list_tools returns the server's `parameters`
   VERBATIM as inputSchema; call_tool validates args shim-side
   (validate_input) and forwards via `anyio.to_thread` (no event-loop block
   during 300s build_run). Defense in depth server-side: `_check_args_shape`
   raises a SELF-CORRECTING ToolError with the usage line
   (`Usage: xdat_get(dat, name, objId)`) before any routing.
2. **`tonumber(string.gsub(...))` passes gsub's 2nd return (substitution
   count) as tonumber's BASE.** 9 sites (GETTBL/SETTBL/GETBTN/SETBTN/
   PLUGADD/PLUGSET/PLUGDEL/PLUGMOVE×2) → parenthesized
   `tonumber((string.gsub(...)))`. rules.md crash rule added + static
   regression test (no unparenthesized pattern may reappear).
3. **`BridgeBusy` is not a `BridgeError` subclass** — _dispatch / journal
   snapshot / map service translated only BridgeError, so a bridge timeout
   escaped as 500. → translate `(BridgeError, BridgeBusy)` everywhere +
   /tools/call last-resort `except Exception` → `{"ok": false, "error":
   "<tool> failed internally: ..."}` (never a 500 to the shim).
4. **Changeset map item showed the journal entry verbatim.** New
   `Journal._map_item`: old = "" (pre-state lives in the backup file),
   new = human summary ("location #1 'P1 Hero Spawn' created in test.scx").
   Journal ENTRY unchanged — rollback keys intact.
5. **The system prompt never stated the target language.** New `[epscript]`
   section (after [first principles], before reference context): epScript
   only; explicit classic-Trigger{} ban; entry functions
   (onPluginStart/beforeTriggerExec/afterTriggerExec, no PreserveTrigger);
   var/const/$U/$L syntax essentials; "search_docs before guessing syntax".
   Intro line now names epScript.
