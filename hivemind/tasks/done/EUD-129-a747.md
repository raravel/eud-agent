## Description
Register and implement the `map_info` READ tool in the Rust tool layer. The schema is NOT yet
registered (only location_write/player_setup schemas exist) and `chk.rs::digest_chk` (EUD-105)
currently has zero callers (dead code). The handler digests the connected source map:
`isom::chk_extract(OpenMapName)` -> `chk::digest_chk(bytes)` -> mode slicing.

- Register `map_info` (mode enum `summary|locations|units|players`, optional `owner`/`unitType`
  filters) as a READ tool (counts one action, no mutation/plan-gate/journal; validate-before-run).
- Resolve the source map path via bridge STATUS `OpenMapName` (bridge_io), spawn-free (in-process
  isom). Slice the digest per mode: summary aggregates only; units mode caps at 200 + truncated
  marker + owner/unitType filters. Include `map.path` + `map.savedAt` (mtime).
- Missing/locked map or unconfigured engine -> a clear ToolError (advisory shape; never a crash).
