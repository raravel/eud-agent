## Description
Wire the already-ported `ProjectMemory` store (src-tauri/src/memory.rs — complete and
unit-tested, currently referenced nowhere outside its own module) into the live app. Three
wiring points, per feature 11 "## memory":

1. **Tauri commands**: add `memory_get` / `memory_save` command handlers and register them in
   the lib.rs `invoke_handler`. The panel already invokes them (EUD-082; panel/src/lib/ipc.ts)
   and currently logs "IPC command failed (memory_get): Command memory_get not found".
   - `memory_get {}` -> `{project, files: {resources, structure, conventions, lessons},
     episodes}` (episodes: last 50, newest first); error when no project is open (STATUS
     project empty).
   - `memory_save {file, content}` -> `{file}`; same file enum + 8 KB cap as `memory_write`;
     atomic write via the store, NO journal entry (user edit, not an agent mutation); error
     on no project / unknown file / oversize content.
   - The project name comes from bridge STATUS via the EUD-147 wiring (depends_on).
2. **Per-turn prompt injection**: lib.rs currently constructs `AgentEngineConfig` with
   `project_memory = None` at startup. Replace the static value with a per-turn refresh so
   `build_system_prompt` / `resume_turn_text` receive a freshly rendered
   `ProjectMemory::render_section(list_reply)` on EVERY turn (first + resumed). Section
   placement is already handled inside engine.rs (`project_memory_section`); pass the LIST
   reply for the staleness annotation when available. Empty project name -> store disabled
   -> "(no project memory)" degradation, never a blocked turn.
3. **Episode recording**: append one episode line at each request finalization point
   (changeset decided / default-accept on next chat / answer-only turn end) per feature 07
   "Episodes" line shape. Append failures are logged and swallowed — memory must never break
   the request flow.

Out of scope (belongs to the broader tool-execution/MCP wiring, not yet planned):
`memory_write` tool execution dispatch and journal item kind `memory`. Do not add them here.
