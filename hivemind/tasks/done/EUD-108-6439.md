## Description
CI builder that re-embeds the ECA corpus with the runtime fastembed pipeline (Decision 10) into a
read-only self-contained BINARY blob `rag/rag-index.bin` in the **`ERAG`/v1** layout
(`magic b"ERAG" | version u32=1 | count u32 | rows...`), emits a sha256, and prepares it as a
GitHub Release asset. The index is parsed with **std only — NO `rusqlite`/sqlite**: it must be
byte-compatible with `src-tauri/src/rag.rs` `write_index`/`load_index` (EUD-109, `INDEX_MAGIC
= b"ERAG"`), so the at-rest artifact and the runtime loader agree exactly. The ECA repo +
its `chromadb_bge` are never modified or imported (rules.md).

NOTE (re-grounded 2026-06-09): the prior body said `index.sqlite` with a BLOB schema — that is
stale. Feature 12 and the shipped `rag.rs` use the `.bin` `ERAG`/v1 format, not sqlite.
