## Description
Cross-task integration bug found by the orchestrator while merging EUD-030 (would otherwise only surface in the editor e2e boot handshake): the bridge spawns the server with `psi.Arguments = "-m eud_agent"` (no data-dir signal), but the server keys heartbeat.txt / server.ready / inbox/outbox entirely off `cfg.data_dir`, and `Config.resolve` receives no CLI/env signal from the bridge spawn — so `data_dir` resolves empty, `server.ready` lands relative to the server cwd (`<repo>\server`), and the bridge (polling `<editor>\Data\agent\server.ready`) never validates readiness. The panel would never navigate.

Fix: in `spawnServer()` pass the agent dir explicitly — `psi.Arguments = '-m eud_agent --data-dir "' .. <agentDir without trailing backslash> .. '"'` (quote the path; strip the trailing `\` so the closing quote is not escaped). `__main__.py` already accepts `--data-dir` (line 35) and routes it through `Config.resolve(cli=...)` with top precedence. Update `server/tests/test_bridge_lifecycle_static.py`'s spawn-block check to REQUIRE the `--data-dir` argument marker (this makes the test fail against the current bridge — the verify-first artifact).
