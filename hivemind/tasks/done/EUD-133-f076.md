## Description
eud-agent links `isom_capi.lib` and `ort_sys` (prebuilt ONNX runtime via `fastembed`) into one
MSVC binary for the first time (started in EUD-128). They require opposite C runtimes:
`isom_capi.lib` (+ folded IsomTerrain/CascLib/StormLib/ICU/MappingCore) is `/MT` + `/GL`, while
`ort_sys 2.0.0-rc.12` is `/MD`. A single MSVC binary allows only one CRT, so `cargo build/test
-p eud-agent` cannot link (measured: default `/MD` -> unresolved `isom_*` LNK2019; crt-static ->
unresolved `ort_sys __imp_*` LNK2019). Rebuild the vendored engine on the dynamic CRT (`/MD`) so
isom and ort share one CRT, keeping the in-process static-link topology (Decision 09). This
unblocks EUD-128.
