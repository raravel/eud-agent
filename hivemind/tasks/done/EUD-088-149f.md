## Context

Live session: a eudplib compile failure ("연결맵에 조건에 맞는 플레이어가 없습니다:
플레이어 종류 Human / 스타트 로케이션을 제대로 배치했는지 확인해보세요" +
python traceback) surfaced only in the editor; the agent had no feedback path.

Two gaps:

1. The system prompt NEVER instructed codex to build after applying code. The
   3-attempt self-fix budget has existed since EUD-057, but nothing triggered
   it — codex applied eps and ended the turn; the user discovered the failure
   by building manually.
2. Parser bug: `_TRACEBACK_DESC_RE` (mirroring the editor's
   BuildErrorHandling.vb:42 regex) only matches when `[Error]` and
   `Traceback (most recent call last):` are on the SAME line. eudplib emits the
   description across MULTIPLE lines — the regex matched nothing and the whole
   human message was dropped (BuildError.message == "", only file/line kept).

## Fixes

1. `engine.py`: new pinned `_BUILD_GUIDE` (`[build]` section, between
   `[epscript]` and `[map locations]`): after applying eps/file changes ALWAYS
   run build_run in the SAME turn; on failure read errors, fix, rebuild
   (server-enforced 3-attempt budget); when spent, STOP and report errors
   verbatim. Map-setup failures (no Human player/start location) are flagged
   as MAP problems, not eps bugs. `build_run` tool description reinforces the
   same rule.
2. `edd_runner.py`: `_TRACEBACK_DESC_RE` now DOTALL + non-greedy — captures
   the full multi-line description up to the NEAREST Traceback marker. This is
   a DELIBERATE divergence from the editor regex (documented in the module
   docstring + features/05); file/line still mirror the editor (first
   traceback frame).
