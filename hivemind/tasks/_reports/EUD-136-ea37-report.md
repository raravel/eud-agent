## Summary
Modified `ci/build_rag_index.rs` so the RAG index builder reads its corpus from an in-repo
directory instead of the ECA checkout. The `--eca <dir>` flag and the `DEFAULT_ECA_DIR` absolute
path constant were removed; a `--corpus <dir>` flag (default repo-relative `ci/corpus`) replaces
them, the `ECA_DIR` env override was renamed `CORPUS_DIR`, and the `Args.eca_dir` field was renamed
`corpus_dir`. Resolution precedence (CLI > env > default) is centralized in a new pure helper
`resolve_corpus_dir`. The JSONL schema, embedding, chunking, and `rag-index.bin`/`.sha256` output
format are unchanged.
