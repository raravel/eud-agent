## Description
Build the panel memory view (spec "Panel: memory view").

- `ws/protocol.ts`: add `memory_get`/`memory_save` client messages and
  `memory`/`memory_saved` server messages to the discriminated unions + type guards.
- `state/store.ts`: memory view state (open flag, files, episodes, dirty flags, active
  tab) + reducers for the new server messages.
- `components/MemoryView.tsx` (NEW): header toggle (lucide `BookText`) opens an overlay in
  the conversation column (PlanView/ChangesetView pattern); four file tabs (resources /
  structure / conventions / lessons) edited with the existing Monaco wiring
  (`panel/src/editor/monaco.ts`, language `markdown`); Save sends `memory_save`,
  `memory_saved` clears the dirty flag; read-only episodes list below.
- `App.tsx`: header button + view wiring; send `memory_get` on open.
- `ChangesetView`: render kind `memory` items with the same diff styling as modified
  files, labeled `memory/<file>`.
- All assets local (no CDN — rules.md).
