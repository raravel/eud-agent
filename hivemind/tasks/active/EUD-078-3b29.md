## Description
Create `server/eud_agent/memory.py`: the ProjectMemory store. Responsibilities (spec
"Storage layout", "Prompt injection", "Episodes"):

- Resolve `<data-dir>/harness/<sanitized-project-name>/` from the config data dir + the
  STATUS project name. Sanitize Windows-invalid characters (`<>:"/\|?*`, control chars) to
  `_`, strip trailing dots/spaces. Empty project name → store is disabled (explicit state).
- Read/write the four markdown files (`resources.md`, `structure.md`, `conventions.md`,
  `lessons.md`): UTF-8 without BOM, atomic temp+replace writes, absent file reads as `""`.
  Enforce the 8,192-byte UTF-8 content cap at write time.
- Append episodes to `episodes.jsonl` (UTF-8 no BOM, one JSON object per line, swallow+log
  append failures); read last N episodes.
- Maintain `meta.json` (`version`, `list_hash`, `list_hash_ts`); helper to compute the
  sha256 of a LIST reply and compare for staleness.
- Render the `[project memory]` section: static instruction block, the four files under
  `## <name>` headings (empty omitted), staleness suffix on `structure` when the hash
  mismatches, `## recent episodes` last-10 one-liners with rejected/partial marked,
  40,000-char cap with the documented truncation order (episodes dropped first, then
  lessons tail-truncated, `memory section truncated` marker), `(no project memory)`
  degradation when disabled/unreadable.
