## Description
Add the `memory_write` MCP tool and its journal integration (spec "MCP tool: memory_write").

- `tools.py`: `ToolSpec("memory_write", "write", ...)` with params `file`
  (enum `resources|structure|conventions|lessons`) and `content` (full replacement text).
  Handler validates enum + 8,192-byte cap (ToolError telling codex to condense), returns a
  ToolError when no project is open. Inject the ProjectMemory store into `ToolLayer`
  (same pattern as the journal injection).
- Plan-gate exemption: `memory_write` does NOT increment the mutation count that trips the
  3-mutation `propose_plan` gate; it DOES count toward the 30-action budget.
- `journal.py`: snapshot before = `{content, existed}`; record after; changeset item kind
  `memory` with target `memory/<file>` and a server-side unified diff old→new; inverse op
  restores old content or deletes the file when `existed=false`. Rollback path wired into
  the existing reverse-seq replay.
- `mcp_shim.py` needs no changes (specs fetched dynamically from `/tools/list`).
