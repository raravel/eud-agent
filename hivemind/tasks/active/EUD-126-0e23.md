## Description
Rebuild `engine.rs` into the agentic turn-loop state machine
(`idle -> triage -> answer | plan_review* -> executing -> changeset_review -> idle`) wiring the IPC v2
surface (EUD-122), the codex app-server transport (EUD-123), the MCP tools (EUD-124), and the journal
(EUD-125). The first `chat` of a session starts a thread; every subsequent `chat` RESUMES it (EUD-064);
`reset` drops the thread id. Resumed turns PREPEND refreshed `[project state]` + project memory +
`[reference context]` before a `[user message]` header (EUD-092) — reuse the existing
`build_system_prompt`/`resume_turn_text`. Emit agent_event/answer/plan/changeset/rollback_result/
progress/error. Enforce triage + plan gating, per-request scoping (fresh request_id per chat, live id at
tool-call time), and cancel. The single-shot `run_instruct`/`InstructOutput`/`unified_diff` seam is
retired (decision 13).
