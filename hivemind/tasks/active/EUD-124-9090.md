## Description
Expose the eud-tools registry as an in-process MCP server attached to the codex thread (decision 13).
Extend the EUD-114 tool layer with the full read/write/flow registry (feature 11): server-side arg
validation before any bridge call, the per-request mutation gate (3rd mutating call without an
approved plan -> tool error directing `propose_plan`), the action budget (30/request) and build
self-fix budget (3), evidence-gate integration (mutating calls rejected until one `search_docs` ran),
and live request-id resolution at tool-call time. MCP tools advertise their param JSON schema verbatim;
result content blocks are plain dicts.
