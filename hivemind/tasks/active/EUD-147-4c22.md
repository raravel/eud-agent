## Description
Replace the placeholder bodies of the `status` and `list` Tauri commands (src-tauri/src/ipc.rs —
"The bridge task replaces this placeholder body") with real file-IPC round-trips through the
existing `bridge_io` client. After this task, `invoke("status")` reports the editor's actual
`{compiling, project}` from `status.txt`, and `invoke("list")` returns the real bridge LIST
file table. This is the prerequisite for project-memory wiring (EUD-148): the project name
that keys the memory store comes from STATUS.

Wiring notes (current code):
- `bridge_io.rs` already implements the protocol (srv-<uuid8>.cmd write, .result poll,
  timeouts, stale cleanup) — wire it, do not rewrite it (import-then-extend).
- The editor install path comes from `config.json` via `config.rs` DataDirs. Unset path,
  absent editor, or stale heartbeat must surface the friendly "editor not connected" error
  contract (feature 11 Edge cases) — never a panic, never a hang without timeout.
- Manage whatever shared bridge state is needed (e.g. `app.manage(...)` in lib.rs setup),
  matching how the engine is managed today.
- rules.md IPC constraints apply: UTF-8 without BOM, delete consumed `.result`, clear stale
  inbox/outbox at startup, 10s timeout extended to 180s while `status.txt` says
  `compiling=true`.
