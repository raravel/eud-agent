## Bug report (user, live panel observation)

Three defects in the agent turn rendering, observed against the live panel:

1. **codex reasoning is invisible.** The SDK streams `item/reasoning/summaryTextDelta` and `item/reasoning/textDelta` notifications (payload field `delta: str` — verified in `server/.venv/Lib/site-packages/openai_codex/generated/notification_registry.py:99-101` and `v2_all.py:2804-2823`), but `_classify_event` in `agent_runner.py` has no branch for them — they fall through to `("event", method)` and the reasoning text is dropped.
2. **The final answer renders faint gray, like reasoning should.** `ConversationLog.tsx` `KIND_CLASS.agent = "text-muted-foreground"` — the agent's answer is the LEAST visible text in the log. User wants the inverse (ChatGPT-style): answer prominent, reasoning dim/secondary.
3. **Raw internal event kinds leak into the UI.** `_classify_event` maps `item/agentMessage/delta` to `("delta", "")` (the delta TEXT in `payload.delta` is discarded), and `AgentStream.tsx` renders `last.detail || last.kind` — so the activity line cycles literal `delta`, `answer`, `token_usage`, `turn_done` strings. Ugly and meaningless to the user.

## Fix design

Server (`agent_runner.py` `_classify_event`):
- `item/reasoning/summaryTextDelta` and `item/reasoning/textDelta` → `("reasoning", <payload.delta>, info)`.
- `item/agentMessage/delta` → `("delta", <payload.delta>, info)` (carry the text instead of "").
- Payload access stays defensive (`getattr`, `or ""`) per the module's existing convention — a missing field degrades to an empty detail, never crashes the turn loop.
- `item/completed` for `rtype == "reasoning"` should NOT emit a noisy generic `item_completed` line (classification may keep the kind but the panel filters; server change optional here — panel filter is the contract).

Panel:
- `AgentStream.tsx` —
  - Accumulate `reasoning` event details into a dim, GPT-style reasoning block (text-muted-foreground, smaller; rendered above/with the activity line; resets per turn like the rest of the stream state).
  - Accumulate `delta` event details into a live answer preview rendered PROMINENTLY (text-foreground) while the turn streams.
  - The "현재:" activity label only ever shows meaningful activity: tool names from `tool_call`/`tool_result`, or a generic 생각 중/응답 작성 중 label. NEVER raw kinds — `delta`, `answer`, `token_usage`, `turn_done`, `item_started`, `item_completed`, `event` must not appear as literal strings in the UI.
- `ConversationLog.tsx` — `agent` kind renders prominent (`text-foreground`); keep `info`/`progress` muted. Reasoning text does NOT go into the log (it lives in the stream component only).
- `App.tsx` — no protocol change: `agent_event{kind, detail}` shape is untouched (features/05 line 57 lists kinds non-exhaustively: "thinking/tool_call/tool_result/turn_done"); adding `reasoning` and text-carrying `delta` is additive.

## Completion criteria
